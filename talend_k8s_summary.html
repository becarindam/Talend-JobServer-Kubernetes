Deployment Process

Step-by-Step Deployment

  1. Build Docker Images: Create base images (JDK, Tomcat) and application images (TAC, JobServer)
  2. Create ConfigMap: kubectl create configmap tac-config --from-file=configuration.properties
  3. Deploy Persistent Volume: kubectl apply -f Talend_Volumes.yaml
  4. Deploy TAC: kubectl apply -f Talend_TAC.yaml
  5. Deploy JobServer: kubectl apply -f Talend_JobServer.yaml
  6. Verify Services: Check pod status and service endpoints

Current Limitations

‚ö†Ô∏è Important Constraints

  • Single Pod Deployment: Only 1 replica per deployment (not horizontally scalable)
  • Stateful Nature: TAC and JobServer require persistent storage
  • No Auto-scaling: Cannot use Kubernetes HPA due to stateful requirements
  • External Database: PostgreSQL runs outside the cluster
  • HA Approach: High availability achieved through multiple separate deployments

Future Enhancements

  • StatefulSet Implementation: Explore Kubernetes StatefulSets for TAC high availability
  • In-Cluster Database: Deploy PostgreSQL within Kubernetes cluster
  • Logging & Monitoring: Implement Fluentd for centralized log collection
  • Security: Add RBAC, network policies, and secrets management
  • CI/CD Integration: Automated build and deployment pipelines
